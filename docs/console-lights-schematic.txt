Gemini said
The documentation for the IBM 1130 indicator lights is primarily located in two specific Systems Reference Library (SRL) manuals. Because the 1130 console is split between the Console Display Panel (the "lamp forest" on the CPU) and the Keyboard/Control Panel (on the desk surface), you need both to get the full picture.

1. IBM 1130 Functional Characteristics (GA26-5881)
This is the primary technical reference. It documents the lights from a hardware and programming perspective.

Location: Look for the section titled "Console Display Panel" or "Indicator Displays."

What’s inside: This section provides a detailed breakdown of the lamp rows, including:

Instruction Address Register (IAR): 16 indicators showing the address of the next instruction.

Storage Data Register (SDR): 16 indicators showing the data being transferred to or from memory.

Accumulator (ACC) and Extension (EXT): Indicators for the primary arithmetic registers.

Condition Indicators: Carry, Overflow, and Zero bits.

Cycle Control: Indicators like Fetch, Execute, and Indirect.

2. IBM 1130 Operating Procedures (GA26-5717)
While the Functional Characteristics manual tells you what the lights mean, the Operating Procedures manual tells you what to do with them.

Location: Look for the section "Console Operator Panel" or "Control Keys and Indicator Lights."

What’s inside: This documents the "Function Lights" located near the keyboard and on the small panel to the right of the CPU, such as:

DISK UNLOCK: Indicates the disk drive is safe to open.

FORMS CHECK: Indicates the console printer is out of paper.

RUN: Indicates the CPU is currently executing instructions.

PARITY/FREEZE: Used for hardware error diagnostics.

Summary Table: Where to Look
Feature	Manual	Typical Page/Section
Register Lights (IAR, ACC, SDR)	Functional Characteristics	Section: Console Display Panel
Status Lights (Run, Wait, Halt)	Functional Characteristics	Section: Indicator Displays
Maintenance/Error Lights	Operating Procedures	Section: Console Operator Panel
I/O Device Lights (Disk, Printer)	Operating Procedures	Section: Readying the System
Pro Tip: If you are using a digital archive like Bitsavers, search for A26-5881 (the original form number). Most versions have a high-resolution diagram of the console layout around page 120 (depending on the revision).

To see the exact documentation for each group, we have to look at the Console Display Panel (the large grid on the CPU) and the Console Keyboard/Control Panel.

Below is the breakdown of these groups as they are documented in the IBM 1130 Functional Characteristics (GA26-5881).

1. The Main Display Panel (CPU)
This is the large, iconic "lamp forest" located on the front of the 1131 Central Processing Unit. It is organized into several distinct rows.

Row 1 & 2: Instruction Address & Storage Data
IAR (Instruction Address Register): 16 indicators. These display the memory address of the next instruction to be performed.

SAR (Storage Address Register): 16 indicators. These show the address of the memory location currently being accessed.

SDR (Storage Data Register): 16 indicators. These display the 16-bit word currently being read from or written to memory.

Row 3 & 4: Arithmetic Registers
ACC (Accumulator): 16 indicators. This is the primary register for arithmetic and logical operations.

EXT (Accumulator Extension): 16 indicators. Used in conjunction with the ACC for double-precision math or multiply/divide operations.

Row 5: The "Status" and "Cycle" Groups
This row is the most complex because it tells you exactly what the "brain" of the computer is doing at any microsecond.

W (Wait): Lights up when the CPU has executed a "Wait" instruction and is idling.

R (Run): Lights up when the CPU is actively executing instructions.

Cycle Indicators: Includes XCT (Execute), FTCH (Fetch), IA (Indirect Address), and DFF (Deferred). These tell you which phase of the instruction cycle the machine is in.

Condition Lights: Includes Carry and Overflow, which reflect the results of the last math operation.

2. Console Keyboard & Functional Lights
These are located on the desk surface, providing status for the peripherals and the "Physical" state of the machine.

Group: Processor State
PARITY: Indicates a memory parity error (a hardware failure).

FREEZE: Lights up when the processor has stopped due to a serious hardware error.

FILE READY: Indicates the internal 2315 disk drive is up to speed and ready for use.

Group: I/O Status
FORMS CHECK: Specifically for the 1132 Printer or the console typewriter; indicates paper jams or "out of paper" states.

DISK UNLOCK: This light glows when it is safe to pull the lever to change the disk cartridge.

3. The Instruction Register (OP Code)
On the far right of the display panel, there is a small vertical or grouped set of lights for the OP Code.

Operation Register: 5 indicators that show the binary code of the instruction currently being performed (e.g., 11000 for a Load instruction).

Where to find the "Official" Map
In the GA26-5881-x manual, you should turn to the Appendix or the Console section. There is a "Console Map" which is a line-art diagram labeled with every single bulb's address.

You are correct. To the right of the four main register rows (IAR, SDR, ACC, EXT) sits a vertical block of indicators known as the Control Display and Operation Register.

These are documented in the IBM 1130 Functional Characteristics (GA26-5881) under the section "Console Display Panel - Right Side." While the register rows show data, these lights show the state of the CPU's internal logic and the specific instruction currently in the "pipeline."

1. The Operation Register (Op Code)
Located at the top-right, these 5 lights represent the machine instruction being executed.

Light	Label	Function
Bit 0	OP 0	Most Significant Bit of the Op Code.
Bit 1	OP 1	
Bit 2	OP 2	
Bit 3	OP 3	
Bit 4	OP 4	Least Significant Bit (e.g., 10001 = Add).
2. The Tag and Format Indicators
Directly below the Op Code are the modifiers that tell the CPU how to handle the instruction's address.

Light	Label	Description
F	Format	On = Long Instruction (2 words); Off = Short Instruction (1 word).
T0	Tag 0	Used for Indexing. If T0 and T1 are both off, no indexing is used.
T1	Tag 1	Binary representation of Index Register 1, 2, or 3.
3. The Cycle Control (The "T-Clock")
Below the Tag bits is a vertical column of lights labeled T0 through T7. These represent the Clock Cycles (Timing pulses) within a single machine cycle.

T0–T7: In a healthy machine, these flash too fast to see during "Run." However, in Single Step or Single Cycle mode, they indicate exactly which sub-step of the instruction the CPU is currently "frozen" in.

OP 0-4	Op Register	Displays the binary value of the current instruction (e.g., Load, Store, Wait).

Where to find the bit-level tables
If you need to know what a specific bit-pattern in the OP CODE group means (e.g., why bits 0 and 3 are lit), you should refer to the "Instruction Formats" table in the same manual (GA26-5881). It provides a hex/binary map for every valid operation.
